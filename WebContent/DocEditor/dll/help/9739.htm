Save Audit

Use the Save Audit option to preserve a record of your edits to Review Cases as Audit XML files in your file structure.

Within an edited Review Case, simply select File | Save Audit to display a Save Audit As window, from which you can specify the drive, directory, and name of the file into which to save the Audit XML

The example audit XML below records the following edits to a Review Case:

  1. Deletion of the word "Motor" from a fragment named 'Regarding'
  2. Insertion of the text "This has been changed" into a fragment named 'Regarding'
  3. Deletion of a fragment named "Letter Body 1"
  4. Insertion of a new fragment with no name, holding the text "New fragment inserted".

Audit XML

These edits were recorded in an Audit XML file as follows:

<?xml version="1.0" ?>

<audit batchname="" batchid="38" transactionno="10">

<audit-document docdef-resid="1890" channel="2" language="-1" jurisdiction="-1" idiom="-1">

<audit-fragment id="13" name="Regarding">

<audit-entry state="Deleted" session="DD\username" time="19/08/2009 13:02" type="Text" value="Motor" />

<audit-entry state="Inserted" session="DD\username" time="19/08/2009 12:58" type="Text" value=". This has been changed." />

</audit-fragment>

<audit-fragment id="1958473474" name="">

<audit-entry state="Inserted" session="DD\username" time="17/08/2009 17:06" type="Fragment" />

<audit-entry state="Inserted" session="DD\username" time="19/08/2009 13:02" type="Text" value="New fragment inserted" />

</audit-fragment>

<audit-fragment id="15" name="Letter Body 1">

<audit-entry state="Deleted" session="DD\username" time="17/08/2009 17:05" type="Fragment" />

</audit-fragment>

</audit-document>

</audit>
