Completing the Document

When all of your edits are completed and any exceptions in your document have been corrected, continue with the normal Workflow.

The next step of the process is determined by the Workflow submission defined within your system.

Where relevant, to submit a Document to Thunderhead, click the Submit icon

The Review Case is resubmitted to Thunderhead and will produce output in each Channel specified by the Batch Config.

Submit in Backwards Compatibility Mode

The job batch collection mode that will be used when the user clicks Submit is set by the API property BatchCollectMode.

By default, Submit uses BatchCollectMode 8, assuming that the destination is Output Processor. If the destination is not Output Processor, then any PDF or HTML or POSTSCRIPT file will be created in intermediate format.

In order for Document Editor to submit work to the server in Backwards Compatibility mode (i.e. not via Output Processor) for creation in other than intermediate format, you must set BatchCollectMode to 0.

For further details, refer to the API section of the Thunderhead document TH Integration - Client Components.pdf, which can be found in your Thunderhead Platform docs directory.
